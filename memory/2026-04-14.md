## What Got Done

### Save to Dashboard (v1.0.6 → v1.0.8)
- Wired "Save to Dashboard" button to Supabase REST API directly
- Extension authenticates via email/password to Supabase (`/auth/v1/token`)
- Popup got full auth UI: login form, logged-in state, sign out
- Added "Don't have an account? Sign up →" link to web dashboard
- **Removed Google OAuth** from extension — Lovable Cloud's managed OAuth doesn't support custom redirect URLs. Email/password works, Google stays on web dashboard only.

### Token Refresh
- When Supabase returns 401 (expired token), extension auto-refreshes using `refresh_token`
- If refresh also fails, clears credentials and shows "Session expired — sign in again"
- Seamless — users won't notice token rotation

### Scan Limits (3/month free)
- Anonymous users: 3 scans per month, then blocked
- Authenticated users: unlimited scans (tracked but not blocked)
- Limit modal with count display + "Sign In" CTA
- Sign-in hint toast: "👆 Click the GhostJob icon in your toolbar to sign in"
- Scan counts stored per-month in `chrome.storage.local`, auto-cleanup old months

### Multi-TLD Careers Page Check
- Was only checking `.com` — missed stripe.io, nonprofits (.org), etc.
- Now checks `.com`, `.io`, `.co`, `.org`, `.net`, `.ai` in parallel
- Tries both `www.company.tld/careers` and `company.tld/careers`
- Quote shows actual domain found (e.g. "stripe.io/careers")

### Duplicate Save Prevention
- Before inserting, queries Supabase for existing `job_url` + `user_id`
- Shows "Job already saved to dashboard" instead of creating duplicates

## Key Decisions
| Decision | Rationale |
|----------|-----------|
| Email/password only in extension | Lovable Cloud's Google OAuth can't add custom redirect URLs for chrome.identity |
| Sign up link opens web dashboard | Web dashboard has Google OAuth; extension just needs email/password |
| Supabase REST API directly from extension | Lovable Cloud is Supabase under the hood; skip the Vercel proxy |
| Authenticated = unlimited scans | Pro tier later; for now signing in removes limits |
| Multi-TLD parallel check | Fast (2.5s timeout each), catches .io/.org companies |
