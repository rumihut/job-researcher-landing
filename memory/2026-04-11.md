### GhostJob Market Research

#### Market Size
- ~43% of job postings may be ghost jobs (2024-2025 research)
- 28-32% across US job boards in 2026
- Job seekers waste ~11+ hours/week on fake listings
- Average application takes 38 min — 15% of search time wasted on ghosts
- $737M lost to fake job offers in US since 2019
- FTC: $500M+ lost to job/employment scams in 2023 alone

#### Competitors
| Competitor | Type | Signals | Free? |
|---|---|---|---|
| JobScamScore | Web checker | 25 red flags | Yes |
| Inteller.ai | AI career advisor | 30+ signals | Freemium |
| Deghost | Web + listings | Unknown | Yes |
| Trouvr | Chrome extension | Unknown | 5 free |
| Lumen | Web checker | Unknown | Yes |
| VantageCV | Web tool | Unknown | Free |
| Subspace | API/Web | 39 signals, validated vs LLMs | Yes |

**Our Edge:** Only tool that injects directly into LinkedIn with one-click button.

#### Key Research Findings (Subspace 1,000-listing study)
- Listing age strongly correlates with ghost probability (30+ days = sharp increase)
- Salary transparency correlates with real jobs
- Reposting patterns = lower quality
- Description quality matters (vague = pipeline posting)

#### Signal Improvements Decided
Added signals for V4:
1. No salary range listed (-4 yellow)
2. Vague/generic description <200 chars substance (-6 red)
3. No team/manager mentioned (-4 yellow)
4. "Various locations" or missing location (-3 yellow)
5. Unrealistic experience combos (-8 red)
6. AI-generated language patterns (-3 yellow)
7. Strengthened urgency weight (-10)
8. Culture buzzwords + red flags contradiction (-4 yellow) — re-added from original

### V4 Code Audit Findings

#### Bugs Found
1. Experience regex weaker in V4 (only 1 pattern vs original's 4)
2. Missing "yellow" signal category (culture buzzwords + red flags)
3. Salary regex too broad (could match IPs/dates)
4. Company extraction bare (1 selector vs original's 4)
5. Location extraction missing entirely
6. Description selector limited (1 vs 4 fallbacks)
7. SPA nav watcher too aggressive (no DOM change throttle)
8. Popup version string says v1.0.0 (manifest says 1.0.4)
9. No visual distinction in score when API fails
10. source field overwritten to 'api' even on local fallback

#### Architecture Decision
- Keep V4 as main — self-contained content script, no background dependency for scanning
- Original `ghostjob-extension/` kept as backup, V4 promoted to active
- V4 folder structure is nested weirdly: `ghostjob-extension-v4/ghostjob-extension/`
