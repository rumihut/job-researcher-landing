## 🧠 Key Learnings

### Technical

**LinkedIn Scraping Reality:**
- Guest API returns **partial HTML** (no `<title>`, no `<head>`)
- Browser sees full HTML, server sees stripped version
- Solution: Extract from **visible content**, not meta tags
- Regex patterns must account for HTML comments (`<!-- -->`)

**X Automation Best Practices:**
- Browser automation > API for reading (no rate limits)
- API > Browser for posting (more reliable)
- Session persistence is key (cookies in `~/.openclaw/auth/`)

### Strategic

**"Dual-Play" Strategy Validated:**
- Manual input form: Works for casual users ✓
- API scraping: Works for automated discovery ✓
